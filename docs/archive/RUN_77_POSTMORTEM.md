## Finding 2: `buildPhaseGatedCatalog` WAS Active During Run #77

The replay snapshot `data/replay-snapshots/run-77.json` stores the exact request bodies. Extracting the system prompt text for the second model request reveals:

```
Your current execution phase is runtimeEnvelope.currentPhase. In this phase, the allowed operations are: listDirectory, readFile.
```

This prose was generated by the line in `buildAgentPrompt`:

```javascript
'Your current execution phase is runtimeEnvelope.currentPhase. In this phase, the allowed operations are: ' + phaseGatedOps.join(', ') + '.'
```

The second request also updated the JSON schema at the end of the system prompt to show only:

```json

```

**Conclusion:** `buildPhaseGatedCatalog` was present in the actual prompt, and the phase-gated operation list **was** present in the prose and in the schema example.

## Root Cause Assessment

Run #77 is **not** evidence that the phase-gated catalog was missing from the prompt. The replay proves the catalog **was** present.

Run #77 **is** evidence that:
1. The model ignored the prose restriction (`listDirectory, readFile` in inspection phase).
2. The `runtimeEnvelope.allowedOperations` JSON blob still contained the full catalog, which may have overridden the prose in the model's reasoning.
3. The workload-profile guidance (`Expected phase pattern: planning → inspection → mutation → verification`) and the explicit `Do not list again in later steps` instruction were also ignored.

The failure is a **model compliance failure**, not a **runtime omission**.
